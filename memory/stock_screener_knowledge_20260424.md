# 주식 스크리너 고도화 및 최적화 전략 (2026-04-24)

## 1. 고속 스캔 엔진 (Fast Filter) 아키텍처
*   **핵심 원칙**: "무거운 분석 전, 가벼운 필터링을 먼저 수행한다."
*   **기술적 구현**: 
    - 전 종목의 일봉을 일일이 다운로드하는 대신, 시장 전체 시세 스냅샷(`pykrx.stock.get_market_cap_by_ticker`)을 사용하여 1차 필터링.
    - 가격 범위, 시가총액, 거래대금, 수급 조건을 시세표 상에서 즉시 검증하여 분석 대상을 10분의 1로 압축.
*   **효과**: 스캔 속도가 비약적으로 향상되어 장중 실시간 대응이 가능해짐.

## 2. 200일 박스권 매집 필터 (Box Squeeze)
*   **전략적 의미**: 주가가 장기간(약 1년, 200거래일) 일정 구간에서 에너지를 응축하며 매집하는 패턴 포착.
*   **수식**: `(최근 200일 최고가 - 200일 최저가) / 200일 최저가 <= 0.30 (30%)`
*   **매매 활용**: 이 박스권 조건에 `정배열`과 `당일 5/10 골든크로스`가 겹치는 시점이 강력한 화산 폭발(급등)의 시작점임.

## 3. 당일 돌파 정밀 타격 (Today Only)
*   **철학**: "주식은 타이밍이다. 어제 터진 것은 이미 늦었다."
*   **로직 구현**:
    - `prev` (어제 종가/지표)와 `last` (오늘 종가/지표)를 엄격하게 비교.
    - 어제는 이동평균선 아래에 있었고, 오늘 비로소 위로 뚫고 올라온 **'최초의 교차점'**만 포착하도록 설계.
*   **장점**: 추격 매수를 방지하고 상승 초입의 가장 신선한 종목만 선별 가능.

## 4. 데이터 수집의 안정성 (Reliability)
*   **영업일 자동 탐색**: 주말이나 장 개시 전에도 정상 작동하도록 최근 7일 내의 마지막 영업일을 자동 추적하는 로직 구현.
*   **엔진 이중화**: `pykrx` 엔진 오류 시 `FinanceDataReader(FDR)`로 자동 전환되는 Fail-safe 시스템 구축.

## 5. UI/UX 개선: 원클릭 체크박스 시스템
*   **사용자 편의성**: 검색이 불편한 멀티셀렉트 대신, 사이드바에 모든 필터를 고정 체크박스로 배치.
*   **전략 조합**: 마우스 클릭 몇 번으로 복잡한 기술적/수급적 조건을 자유롭게 조합하여 나만의 필승 전략을 즉시 실행 가능.
